# Website – Overview

## Description

The **Website** module is used to manage all content and features displayed on the site.
Administrators can configure company information, static pages, dynamic content, and handle interactions from external users through the management panel.

This module serves as the control center to ensure the website's appearance and content are always up-to-date, consistent, and aligned with business needs.

---

## Purpose

-   Provide a single interface to manage all parts of the website.
-   Make it easier for administrators to add, modify, and delete content.
-   Offer interaction channels between the company and users (such as career forms, collaboration requests, contact messages).
-   Support SEO and social media integration to improve website discoverability.

---

## Key Concepts

-   **Dashboard** → Summary of main information and recent activities in the Website module.
-   **Category** → Content grouping structure (e.g., products, articles, posts).
-   **Post** → Articles or news published on the website.
-   **Content** → Other dynamic content that can be managed (e.g., content lists by type).
-   **Page** → Static pages such as _Terms & Conditions_ or _Privacy Policy_.
-   **Product & Store** → Information about available products and stores.
-   **Career & Career Applicants** → Open job vacancies and applicant lists.
-   **Investor Documents** → Public documents for investors.
-   **FAQ** → Frequently asked questions from users.
-   **Collaboration Request** → Forms for external parties to request collaboration.
-   **Contact Messages** → Messages sent through the contact form.
-   **Slider** → Banners or sliders displayed on the homepage.
-   **SEO** → Metadata configuration to optimize the website for search engines.
-   **Instagram** → Instagram feed integration.
-   **Profile** → Company profile information displayed on the website.

---

## Basic Flow

1. **Content & Structure**

    - Admin creates categories, posts, static pages, products, or FAQs as needed.
    - Content can be managed in multiple languages (multi-locale).

2. **User Interaction**

    - External users can send messages, apply for jobs, or submit collaboration requests.
    - Incoming data will be recorded in the relevant module for further management.

3. **Media & Appearance**

    - Admin adds sliders, product images, or avatars to enhance the website's appearance.
    - Display preferences can be set to maintain brand consistency.

4. **Optimization**

    - Admin configures SEO metadata and social media integration to increase content reach.

---

## Notes

-   Use **Category** to maintain consistent content structure.
-   Ensure each static **Page** contains content that meets legal and public information requirements.
-   **Career Applicants** can only be managed from data submitted through the application form.
-   **Media** is currently limited to image files; video or large document uploads are not supported.
-   Always complete metadata in **SEO** for each page to optimize search performance.
-   Use the **Website Dashboard** as the starting point for monitoring this module's activities.
